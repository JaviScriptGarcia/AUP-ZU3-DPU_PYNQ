# v0.1-pynq-vanilla — Validation log

Date: 2026-06-05.

## What is being claimed

The official PYNQ 3.1.1 image for the AUP-ZU3 8 GB variant boots
unmodified on our board, with network connectivity and SSH access
from the host PC.

## How it was verified

### Boot sequence (UART, 115200 8N1 on /dev/ttyUSB1)

- FSBL banner: `Zynq MP First Stage Boot Loader, Release 2024.1`
- ATF: `BL31: v2.10.0`
- U-Boot 2024.01, `Model: AUP-ZU3 v1.0`
- DDR detection: `2 GiB (effective 8 GiB)` (8 GB visible)
- Kernel 6.6.10-xilinx-v2024.1 booting on 4x Cortex-A53
- Init: systemd 249, PYNQ Linux 22.04
- Auto-login as `xilinx`, prompt reached in ~13 s

### Filesystem expansion

On second boot, `Resize Filesystem on SD card` ran and the rootfs
grew from 9 GB (image) to 116 GB (full microSD).

### Network

USB Ethernet adapter (Realtek RTL8152) plugged into a USB3-A port
on the board, Ethernet cable to the local router.

- `eth0` link: `carrier on`
- DHCP lease: `192.168.1.44`
- `ping 8.8.8.8`: 0% loss, ~15 ms
- `ping google.com`: DNS resolves, 0% loss, ~18 ms

### Access from host

```
ssh xilinx@192.168.1.44     # password: xilinx
```

Inside the board:

```
$ python3 -c "import pynq; print(pynq.__version__)"

$ cat /etc/os-release | head -3

```

## Inconsequential observations during boot

- `xilinx-dp-snd-codec ... probe failed with error -22` — DisplayPort
  audio codec probe failure. No DisplayPort connected, harmless.
- `zocl-drm axi:zyxclmm_drm: error -ENXIO: IRQ index 8 not found` —
  the zocl driver (XRT in-kernel) looks for the DPU IRQ. No DPU
  overlay loaded yet, harmless until v0.2.
- `rtc: setting system clock to 2105-12-20` and later `2035-10-16` —
  the RTC has no battery backup. NTP will correct it when there is a
  reachable server.
- A second IP (`192.168.2.99/24`) is also assigned to `eth0`. Likely
  leftover static configuration from the image; harmless because the
  router serves `192.168.1.x`.

## Conclusion

**v0.1-pynq-vanilla is met.** Vanilla PYNQ boots cleanly on the
AUP-ZU3 8 GB and is reachable over SSH from the host. The board is
ready to receive DPU overlays in v0.2.
